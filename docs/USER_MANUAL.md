## 系统概述

### 系统简介

物流供应链可视化面板是一个综合性的物流管理系统，提供仓储管理、客户管理、供应商管理、订单管理、库存管理、运输管理等核心功能，支持数据可视化大屏展示。

### 系统特点

- **可视化大屏**: 直观的数据展示，实时监控物流运营状况
- **模块化管理**: 清晰的功能模块划分，操作简单便捷
- **数据实时更新**: 支持数据的实时更新和同步
- **多维度分析**: 提供多种数据分析维度和报表
- **响应式设计**: 支持多种设备和屏幕尺寸

### 适用场景

- 物流仓储企业
- 供应链管理公司
- 第三方物流服务商
- 电商物流部门

## 功能模块

### 1. 仓储管理

#### 功能说明

仓储管理模块提供仓库信息的全生命周期管理，包括仓库的创建、编辑、删除和查询功能。

#### 主要功能

- **仓库列表**: 查看所有仓库的基本信息
- **仓库详情**: 查看单个仓库的详细信息
- **新增仓库**: 创建新的仓库记录
- **编辑仓库**: 修改仓库的基本信息
- **删除仓库**: 删除不再使用的仓库

#### 数据字段

| 字段名称 | 字段说明 | 是否必填 | 示例 |
|---------|---------|---------|------|
| 仓库名称 | 仓库的名称 | 是 | 上海一号仓 |
| 仓库地址 | 仓库的物理地址 | 是 | 上海市浦东新区张江高科技园区 |
| 联系人 | 仓库负责人 | 是 | 张三 |
| 联系电话 | 联系电话 | 是 | 13800138000 |
| 容量 | 仓库容量 | 是 | 10000 |
| 状态 | 仓库状态 | 是 | 正常 |

#### 操作步骤

**新增仓库**:

1. 进入仓储管理页面
2. 点击"新增仓库"按钮
3. 填写仓库信息
4. 点击"保存"按钮

**编辑仓库**:

1. 进入仓储管理页面
2. 找到要编辑的仓库
3. 点击"编辑"按钮
4. 修改仓库信息
5. 点击"保存"按钮

**删除仓库**:

1. 进入仓储管理页面
2. 找到要删除的仓库
3. 点击"删除"按钮
4. 确认删除操作

## 常见问题

### Q1: 系统无法访问

**A**: 请检查以下内容：

1. 确认系统服务是否正常启动
2. 检查网络连接是否正常
3. 确认访问地址是否正确
4. 检查防火墙设置

### Q2: 数据保存失败

**A**: 请检查以下内容：

1. 确认必填字段是否已填写
2. 检查数据格式是否正确
3. 确认数据库连接是否正常
4. 查看系统日志获取详细错误信息

### Q3: 数据显示不正确

**A**: 请检查以下内容：

1. 刷新页面重新加载数据
2. 清除浏览器缓存
3. 检查筛选条件是否正确
4. 确认数据库中的数据是否正确

### Q4: 导出功能无法使用

**A**: 请检查以下内容：

1. 确认浏览器是否允许下载
2. 检查导出文件格式是否支持
3. 确认数据量是否过大
4. 检查网络连接是否正常

### Q5: 数据大屏显示异常

**A**: 请检查以下内容：

1. 刷新页面重新加载数据
2. 检查浏览器兼容性
3. 确认数据源是否正常
4. 检查网络连接是否正常

## 系统管理

### 系统配置

#### 数据库配置

**本地部署配置**:

编辑 `backend/src/main/resources/application-dev.yml`:

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/logistics_db
    username: root
    password: root
```

**Docker部署配置**:

编辑 `deploy/docker-compose.yml`:

```yaml
environment:
  MYSQL_ROOT_PASSWORD: root
  SPRING_DATASOURCE_PASSWORD: root
```

#### 前端配置

编辑 `FrontEnd/.env.development`:

```env
VITE_API_BASE_URL=http://localhost:8080/api
```

### 系统监控

#### 查看系统日志

**本地部署**:

```bash
# 后端日志
tail -f backend/logs/logistics-dashboard.log

# 前端日志（浏览器控制台）
```

**Docker部署**:

```bash
# 查看所有服务日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f frontend
docker-compose logs -f mysql
```

- **内存使用率**: 监控系统内存使用情况

### 系统维护

#### 定期备份

建议每周进行一次数据备份：

```bash
# 本地部署
mysqldump -u root -p logistics_db > backup_$(date +%Y%m%d).sql

# Docker部署
docker-compose exec mysql mysqldump -uroot -proot logistics_db > backup_$(date +%Y%m%d).sql
```

#### 数据清理

定期清理过期数据和日志文件：

```bash
# 清理过期日志
find backend/logs -name "*.log.*" -mtime +30 -delete

# 清理Docker日志
docker system prune -a
```

#### 系统更新

**更新代码**:

```bash
# 拉取最新代码
git pull

# 重新构建并启动（Docker部署）
cd deploy
docker-compose up -d --build

# 重新启动服务（本地部署）
cd backend
mvn clean package
java -jar target/*.jar
```
