# 项目启动器使用指南

## 概述

项目启动器是一个可视化的图形界面工具，用于简化物流仓储大数据面板项目的环境验证、配置管理和服务启动流程。

## 功能特性

### 1. 环境验证
- 自动检测 Java、Node.js、MySQL、Docker、Maven、Nginx 等依赖
- 显示各组件的版本信息
- 提供详细的环境检查报告

### 2. 配置管理
- 可视化配置后端和前端路径
- 配置服务端口（后端、前端、MySQL）
- 配置数据库连接信息
- 配置持久化保存

### 3. 服务控制
- 一键启动/停止单个服务
- 一键启动/停止所有服务
- 实时显示服务运行状态
- 支持服务重启

### 4. 日志监控
- 实时显示服务运行日志
- 按服务类型过滤日志
- 日志级别颜色区分（INFO、SUCCESS、ERROR、WARNING）
- 支持日志清空和刷新

## 快速开始

### 方法一：使用启动脚本（推荐）

1. 双击运行 `启动器.bat` 文件
2. 启动器将自动打开图形界面

### 方法二：使用 Python 命令

```bash
python launcher.py
```

## 使用说明

### 环境验证标签页

1. 启动器会自动检查所有依赖环境
2. 查看各组件的状态：
   - ✓ 绿色：已安装且版本正确
   - ⚠ 橙色：警告（可选组件）
   - ✗ 红色：未安装或版本不兼容
3. 点击"重新检查环境"按钮可重新检查

### 配置管理标签页

1. **路径配置**
   - 后端路径：指向 backend 目录
   - 前端路径：指向 FrontEnd 目录
   - 点击"浏览"按钮选择路径

2. **端口配置**
   - 后端端口：默认 8080
   - 前端端口：默认 3000
   - MySQL 端口：默认 3306

3. **数据库配置**
   - 用户名：数据库用户名
   - 密码：数据库密码
   - 数据库名：数据库名称

4. 点击"保存配置"按钮保存设置

### 服务控制标签页

1. **单个服务控制**
   - 点击"启动"按钮启动对应服务
   - 点击"停止"按钮停止对应服务
   - 状态标签显示服务运行状态

2. **快捷操作**
   - 一键启动所有服务：按顺序启动 MySQL、后端、前端
   - 一键停止所有服务：停止所有运行中的服务
   - 重启所有服务：先停止再启动所有服务

3. **访问地址**
   - 点击"打开前端页面"按钮可直接在浏览器中打开应用

### 运行日志标签页

1. **日志来源选择**
   - 全部：显示所有服务日志
   - 后端：仅显示后端服务日志
   - 前端：仅显示前端服务日志
   - MySQL：仅显示 MySQL 服务日志

2. **日志操作**
   - 清空日志：清除当前显示的所有日志
   - 刷新日志：重新加载日志信息

3. **日志颜色说明**
   - 蓝色：INFO 信息
   - 绿色：SUCCESS 成功
   - 红色：ERROR 错误
   - 橙色：WARNING 警告

## 环境要求

### 必需组件
- **Python 3.7+**：运行启动器
- **Java 8+**：运行后端服务
- **Node.js 16+**：运行前端服务
- **MySQL 8.0+**：数据库服务

### 可选组件
- **Docker**：使用 Docker 运行 MySQL
- **Maven**：使用 Maven 构建和运行后端
- **Nginx**：生产环境部署

## 常见问题

### 1. 启动器无法启动

**问题**：双击 `启动器.bat` 无反应

**解决方案**：
- 确认已安装 Python 3.7+
- 在命令行中运行 `python --version` 检查
- 将 Python 添加到系统 PATH 环境变量

### 2. 环境检查失败

**问题**：某些组件显示为未安装

**解决方案**：
- 安装对应的组件
- 确认组件已添加到系统 PATH
- 重启启动器并重新检查

### 3. 服务启动失败

**问题**：点击启动按钮后服务无法启动

**解决方案**：
- 检查端口是否被占用
- 查看运行日志标签页获取详细错误信息
- 确认配置路径是否正确
- 检查数据库连接信息是否正确

### 4. MySQL 启动失败

**问题**：MySQL 服务启动失败

**解决方案**：
- 确认 Docker 已安装并运行
- 检查 MySQL 端口是否被占用
- 查看日志了解具体错误
- 可手动启动 MySQL 服务

### 5. 后端启动失败

**问题**：后端服务启动失败

**解决方案**：
- 确认 Java 已正确安装
- 检查后端路径配置是否正确
- 查看运行日志获取详细错误
- 确认数据库服务已启动
- 检查数据库连接配置

### 6. 前端启动失败

**问题**：前端服务启动失败

**解决方案**：
- 确认 Node.js 已正确安装
- 检查前端路径配置是否正确
- 确认已执行 `npm install` 安装依赖
- 查看运行日志获取详细错误

## 配置文件

启动器配置保存在 `launcher_config.json` 文件中，包含以下内容：

```json
{
  "backend_path": "后端路径",
  "frontend_path": "前端路径",
  "backend_port": 8080,
  "frontend_port": 3000,
  "mysql_port": 3306,
  "mysql_user": "root",
  "mysql_password": "root123",
  "database_name": "logistics_db"
}
```

## 高级功能

### 使用 Docker Compose

如果已配置 Docker Compose，可以使用以下命令启动所有服务：

```bash
cd deploy
docker-compose up -d
```

### 手动启动服务

**启动 MySQL（Docker）**：
```bash
docker run -d --name logistics-mysql \
  -e MYSQL_ROOT_PASSWORD=root123 \
  -e MYSQL_DATABASE=logistics_db \
  -p 3306:3306 \
  mysql:8.0
```

**启动后端**：
```bash
cd backend
java -jar target/logistics-dashboard-1.0.0.jar
```

**启动前端**：
```bash
cd FrontEnd
npm run dev
```
